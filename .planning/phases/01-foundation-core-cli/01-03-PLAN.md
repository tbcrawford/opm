  <action>
    **cmd/init.go** — `opm init`:
    Implement the crash-safe 3-step sequence (per D-16). Logic (in the RunE, delegating to internal packages):

    ```
    opencodeDir = paths.OpencodeConfigDir()
    opmDir = paths.OpmDir()
    defaultProfileDir = paths.ProfileDir("default")

    Step 0 — Idempotency checks:
      st = symlink.Inspect(opencodeDir)
      if st.IsSymlink && strings.HasPrefix(st.Target, paths.ProfilesDir()):
        error: "opm is already initialized. Active profile: <basename(target)>"
      if defaultProfileDir exists AND opencodeDir is a symlink pointing to it:
        same error (resumed after crash, already done)

    Detect partial crash state:
      if defaultProfileDir exists AND not (opencodeDir is symlink to it):
        // Step 2 completed but step 3 didn't — resume from step 3
        goto step3

    Normal init path:
      s = newStore()
      s.Init()  // creates ~/.config/opm/profiles/

      if st.IsDir (real opencode dir exists):
        // Step 1: create temp symlink
        tmpSym = opencodeDir + ".opm-new"
        symlink.SetAtomic(defaultProfileDir, tmpSym)
        // Step 2: move the real directory to profiles/default
        os.Rename(opencodeDir, defaultProfileDir)
        // Step 3: install the real symlink atomically
        os.Rename(tmpSym, opencodeDir)
      else:
        // Fresh machine: no opencode dir exists
        os.MkdirAll(defaultProfileDir, 0o755)
        symlink.SetAtomic(defaultProfileDir, opencodeDir)

      s.SetCurrent("default")
      print: "Initialized opm. Active profile: default"
    ```

    No --force flags for init in Phase 1. If opencodeDir is a non-opm foreign symlink, error with:
    `~/.config/opencode is an unrecognized symlink. Back it up and remove it, then run 'opm init' again.`

    Do NOT attach managedGuard to init (init is the bootstrap command — it runs before managed state exists).

    ---

    **cmd/context_create.go** — `opm context create <name>`:
    ```
    Args: cobra.ExactArgs(1)
    PersistentPreRunE: managedGuard (attach in init())
    RunE:
      name = args[0]
      s = newStore()
      if err := s.CreateProfile(name); err != nil:
        return err  // includes validation error if name invalid (per POLISH-02)
      fmt.Printf("Created context %q\n", name)
    ```

    ---

    **cmd/context_use.go** — `opm context use <name>`:
    ```
    Args: cobra.ExactArgs(1)
    PersistentPreRunE: managedGuard
    RunE:
      name = args[0]
      s = newStore()
      profileDir, err = s.GetProfile(name)
      if err: return err  // "context "foo" does not exist"
      if err := symlink.SetAtomic(profileDir, paths.OpencodeConfigDir()); err != nil:
        return err
      if err := s.SetCurrent(name); err != nil:
        return err
      fmt.Printf("Switched to context %q\n", name)
    ```

    Register each command in its file's `init()`:
    - `init.go`: `rootCmd.AddCommand(initCmd)`
    - `context_create.go`: `contextCmd.AddCommand(createCmd)`
    - `context_use.go`: `contextCmd.AddCommand(useCmd)`

    Error output: cobra prints errors to stderr by default. Verify with `cmd.SilenceUsage = true` and `cmd.SilenceErrors = false` on rootCmd to avoid double-printing.

    Run `go build ./...` after each file.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI → cmd/ | Profile names and flags arrive from user; cmd/ validates via store.ValidateName before use |
| cmd/ → filesystem | init modifies ~/.config/opencode — a critical system path; crash-safety is required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `opm init` crash mid-migration | mitigate | 3-step sequence: temp symlink → rename dir → atomic swap; each step is recoverable on restart |
| T-03-02 | Elevation of Privilege | `context create` with traversal name | mitigate | Delegated to store.CreateProfile which calls ValidateName first (D-17) |
| T-03-03 | Denial of Service | `opm init` on already-managed symlink | mitigate | Idempotency check exits early with clear error rather than re-running migration |
| T-03-04 | Tampering | `context use` on non-existent profile | mitigate | store.GetProfile validates existence before SetAtomic is called |
| T-03-05 | Spoofing | Running context commands before `opm init` | mitigate | managedGuard PersistentPreRunE blocks all context subcommands if not opm-managed (D-08/D-09) |
</threat_model>
