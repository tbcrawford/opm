  <name>Task 1: Add Dangling field to store.Profile and dangling detection to ListProfiles</name>
  <files>internal/store/store.go, internal/store/store_test.go</files>
  <behavior>
    - Profile struct gains `Dangling bool` field
    - ListProfiles: after scanning profiles/ dir, checks if the active symlink (via Readlink) points to a dir NOT in the scanned list
    - If so: synthesizes a Profile{Name: basename, Path: target, Active: true, Dangling: true} and includes it in the sorted results
    - If the symlink itself is already dangling (os.Lstat on target shows not-exist), Dangling is true
    - Existing profiles (from dir scan) have Dangling: false always (they exist by definition of ReadDir)
  </behavior>
  <action>
    In `internal/store/store.go`:

    1. Add `Dangling bool` to Profile struct:
    ```go
    type Profile struct {
        Name    string
        Path    string
        Active  bool
        Dangling bool
    }
    ```

    2. In `ListProfiles`, after building the profiles slice from ReadDir and before returning:
    ```go
    // Check for dangling active profile (dir deleted while symlink remains).
    activeTarget, readlinkErr := os.Readlink(s.opencodeDir)
    if readlinkErr == nil {
        activeName := filepath.Base(activeTarget)
        // Is the active name already in the scanned list?
        found := false
        for _, p := range profiles {
            if p.Name == activeName {
                found = true
                break
            }
        }
        if !found {
            // Active symlink points to a profile dir that no longer exists.
            profiles = append(profiles, Profile{
                Name:     activeName,
                Path:     activeTarget,
                Active:   true,
                Dangling: true,
            })
            // Re-sort after appending.
            sort.Slice(profiles, func(i, j int) bool {
                return profiles[i].Name < profiles[j].Name
            })
        }
    }
    ```

    Add to `internal/store/store_test.go`:
    - `TestListProfiles_DanglingActive`: create store, create "work" profile, symlink opencodeDir → work, then os.RemoveAll(work dir), call ListProfiles, assert one profile returned with Dangling=true, Active=true, Name="work"
    - `TestListProfiles_HealthyActive_NotDangling`: normal case — active profile exists, Dangling must be false
  </action>
  <verify>
    <automated>go test ./internal/store/... -v -count=1 -run "TestListProfiles"</automated>
  </verify>
  <done>Profile.Dangling field exists; TestListProfiles_DanglingActive passes; all existing store tests pass.</done>

      if p.Dangling { marker = "!" }
      else if p.Active { marker = "*" }
      else { marker = " " }
    Display name: if p.Dangling { name = p.Name + " (missing)" } else { name = p.Name }
  </behavior>
  <action>
    Update `runContextLs` in `cmd/context_ls.go`:
    ```go
    for _, p := range profiles {
        marker := " "
        displayName := p.Name
        if p.Dangling {
            marker = "!"
            displayName = p.Name + " (missing)"
        } else if p.Active {
            marker = "*"
        }
        fmt.Fprintf(w, "%s %s\n", marker, displayName)
    }
    ```
